<h1>What's new in R3 | With links to User Guide</h1>
<p>This list details the changes users can expect in the upcoming SLS December
2025 Release (Release 3), which will be rolled out on 4 December 2025.
Please note that SLS will be unavailable from Tuesday, 2 December 2025
to Thursday, 4 December 2025 evening to facilitate this update.</p>
<h2>Features for All Users</h2>
<ol data-tight="true" class="tight">
<li>
<p>Annotation Enhancements</p>
<ol data-tight="true" class="tight">
<li>
<p>Teachers and students will see a consistent interface when creating and
editing notes, annotations, and discussion posts on SLS.</p>
</li>
<li>
<p>Annotation cards now display the media objects they reference, making
it easier for users to see the context of their notes.</p>
</li>
</ol>
</li>
<li>
<p>Rich Text Editor</p>
<ol data-tight="true" class="tight">
<li>
<p>Teachers and students can now use cut, copy, and paste options on the
toolbar and right-click context menu in addition to keyboard shortcuts.</p>
</li>
<li>
<p>Teachers and students can now use additional formatting options such as
underline, strikethrough, superscript, and subscript in most Rich Text
Editors on SLS.</p>
</li>
<li>
<p>The toolbar for Rich Text Editors has been reorganised and can now be
collapsed on desktop screens. It is collapsed by default on mobile screens.</p>
</li>
</ol>
</li>
<li>
<p>UI/ UX Enhancements</p>
<ol data-tight="true" class="tight">
<li>
<p>Icons and illustrations on SLS have been refreshed and AI-enabled features
can be identified by an orange sparkle.</p>
</li>
<li>
<p>Subpages have increased height and width for desktop and tablet, improving
content visibility.</p>
</li>
<li>
<p>Module tags and content maps are now displayed on module covers.</p>
</li>
<li>
<p>Shaded grey boxes framing images and videos have been removed. The frame
now only appears when users hover over the image, improving the viewing
experience.</p>
</li>
<li>
<p>Table displays have been refined to improve clarity and visibility of
content.</p>
</li>
</ol>
</li>
<li>
<p><a href="https://www.learning.moe.edu.sg/teacher-personalise-sls-homepage/" rel="noopener nofollow" target="_blank">Homepage Enhancements</a>
</p>
<ol data-tight="true" class="tight">
<li>
<p>A "Continue Where You Left Off" section has been added on the homepage,
allowing teachers and students to quickly return to recently accessed modules,
assignments and templates.</p>
</li>
<li>
<p>The homepage now supports alt-text for screen reader compatibility and
keyboard navigation using the "Tab" key.</p>
</li>
<li>
<p><a href="https://www.learning.moe.edu.sg/teacher-personalise-sls-homepage/" rel="noopener nofollow" target="_blank">Teachers</a>,
<a href="https://www.learning.moe.edu.sg/admin-personalise-sls-homepage/" rel="noopener nofollow" target="_blank">SLS School Administrators</a>and <a href="https://www.learning.moe.edu.sg/student-personalise-sls-homepage/" rel="noopener nofollow" target="_blank">students</a> will
see an updated list of shortcuts on the homepage, reflecting the latest
changes.</p>
</li>
<li>
<p>Teachers and students will see a refreshed Account landing page that combines
key functions (password, linked accounts, email/notification settings,
avatar, profile, followed subjects, switch to student account).</p>
</li>
<li>
<p>Teachers and SLS School Administrators have a new full-sized landing page
that consolidates all admin functions.</p>
</li>
<li>
<p>Teachers with the External Content Developers (ECD) flag will no longer
see the "Switch to Student Account" option.</p>
</li>
</ol>
</li>
<li>
<p>Notifications Enhancements</p>
<ol data-tight="true" class="tight">
<li>
<p><a href="https://www.learning.moe.edu.sg/teacher-user-guide/notify/about-notifications/" rel="noopener nofollow" target="_blank">Teachers</a> and
<a href="https://www.learning.moe.edu.sg/student-user-guide/notify/about-notifications/" rel="noopener nofollow" target="_blank">students</a>can choose their preferred assignment sub-categories for email
notifications, giving them greater control over the notifications they
receive.</p>
</li>
<li>
<p>Teachers and students can view their latest notifications under the notification
bell and mark them as read directly from the notification panel.</p>
</li>
<li>
<p>Links in email notifications are now clickable for direct access.</p>
</li>
<li>
<p>The filters on the notification listing page have been updated to help
users find relevant notifications more easily.</p>
</li>
</ol>
</li>
<li>
<p>Profile Live Indicator</p>
<ol data-tight="true" class="tight">
<li>
<p>Teachers and students can see a green dot on user avatars when users are
active on SLS.</p>
</li>
</ol>
</li>
<li>
<p><a href="https://www.learning.moe.edu.sg/login-troubleshooting/get-help/contact-sls-helpdesk/" rel="noopener nofollow" target="_blank">Help Widget</a>
</p>
<ol data-tight="true" class="tight">
<li>
<p>Teachers and students can access the User Guide via a help widget available
on all SLS pages, except the login page.</p>
</li>
</ol>
</li>
</ol>
<h2>Features for Teachers</h2>
<ol data-tight="true" class="tight">
<li>
<p>Search</p>
<ul data-tight="true" class="tight">
<li>
<p><a href="https://www.learning.moe.edu.sg/teacher-user-guide/organise/star-resources/" rel="noopener nofollow" target="_blank">Starred Resources</a>:</p>
<ol data-tight="true" class="tight">
<li>
<p>Teachers can search for starred resources by title on the Starred Resources
page.</p>
</li>
<li>
<p>Teachers can search for starred resources by title when adding content
from Starred Resources.</p>
</li>
</ol>
</li>
<li>
<p><strong>MOE Library</strong>:</p>
<ol data-tight="true" class="tight">
<li>
<p>Teachers can now search for SLS Professional Development modules in the
MOE Library that were previously visible only in their student account.
Selecting these modules will automatically switch them to their student
account.</p>
</li>
</ol>
</li>
</ul>
</li>
<li>
<p>Authoring</p>
<ul data-tight="true" class="tight">
<li>
<p><a href="https://www.learning.moe.edu.sg/teacher-user-guide/author/add-rubrics-to-audio-and-free-response-questions/" rel="noopener nofollow" target="_blank">Rubrics Enhancements</a>:</p>
<ol data-tight="true" class="tight">
<li>
<p>Teachers can search for rubrics, star rubrics, and add rubrics from search
or Starred Resources to Free-Response Questions (FRQ) and Audio-Response
Questions (ARQ).</p>
</li>
<li>
<p>Existing rubrics will automatically inherit section tags for searchability.</p>
</li>
</ol>
</li>
<li>
<p><strong>Pagination Enhancements</strong>:</p>
<ol data-tight="true" class="tight">
<li>
<p>Teachers can add links directly to specific pages within sections or activities
in modules.</p>
</li>
<li>
<p>Teachers can move components or pages to reorder them within modules.</p>
</li>
</ol>
</li>
</ul>
</li>
<li>
<p>Assignments</p>
<ul data-tight="true" class="tight">
<li>
<p><strong>Half Marks</strong>:</p>
<ol data-tight="true" class="tight">
<li>
<p>Teachers can now award half marks for Free-Response Questions (FRQ) and
Audio-Response Questions (ARQ) in Teacher-Marked Quizzes (TMQ) and standalone
questions.</p>
</li>
<li>
<p>HTML5 and Feedback Assistant - Mathematics (FA-Math) can accommodate half
marks.</p>
</li>
</ol>
</li>
<li>
<p><strong>Editing Attempted Questions</strong>:</p>
<ol data-tight="true" class="tight">
<li>
<p>Teachers can now edit Multiple-Choice Questions (MCQ), Multi-Response
Questions (MRQ), Audio-Response Questions (ARQ), Fill-in-the-Blanks Questions,
Click-and-Drop Questions, Error-Editing Questions, even if they are part
of a Multiple-Part Question (MPQ), in assignments or resources even after
students have attempted them.</p>
</li>
<li>
<p>Teachers and students can view the last edited date and time of a question
if the assignment or resource was edited after a student attempt.</p>
</li>
<li>
<p>This feature is not applicable for assessments.</p>
</li>
</ol>
</li>
<li>
<p><strong>Assignment Sharing and Study Buddy</strong>:</p>
<ol data-tight="true" class="tight">
<li>
<p>Owners and co-teachers can set reminders to enable sharing permissions
upon assignment due dates.</p>
</li>
<li>
<p>Teachers can turn off sharing permissions if needed.</p>
</li>
<li>
<p>This feature is not applicable for assessments.</p>
</li>
</ol>
</li>
<li>
<p><strong>Print-Friendly Worksheet with Responses</strong>:</p>
<ol data-tight="true" class="tight">
<li>
<p>For completed assignments, teachers can allow students to access a print-friendly
version of the assignment with their responses.</p>
</li>
</ol>
</li>
</ul>
</li>
<li>
<p>Authoring Copilot (ACP) Enhancements</p>
<ol data-tight="true" class="tight">
<li>
<p>Teachers can use Authoring Copilot (ACP) to <a href="https://www.learning.moe.edu.sg/teacher-user-guide/author/add-rubrics-to-audio-and-free-response-questions/" rel="noopener nofollow" target="_blank">generate new rubrics</a> or
populate empty fields in existing rubrics for FRQs.</p>
</li>
<li>
<p>Teachers can use ACP to <a href="https://www.learning.moe.edu.sg/generate-interactives-with-authoring-copilot/" rel="noopener nofollow" target="_blank">Generate a HTML5 interactive</a>,
and refine it further with instructions.</p>
</li>
<li>
<p>Teachers can select predetermined recipes (e.g., simple game, simple simulation)
or customise their own instructions, and add knowledge base(s) for ACP
to reference when generating interactives.</p>
</li>
<li>
<p>Teachers can use <a href="https://www.learning.moe.edu.sg/generate-images-with-authoring-copilot/" rel="noopener nofollow" target="_blank">ACP to generate images</a>.
They can choose from 3 generated images with each request.</p>
</li>
<li>
<p>Teachers can generate images for module/section/discussion feature image,
gamification components and Click-and-Drop questions.</p>
</li>
<li>
<p>Teachers can use <a href="https://www.learning.moe.edu.sg/gamify-a-module-with-authoring-copilot/" rel="noopener nofollow" target="_blank">use ACP to generate game stories and collectibles</a>.
Generated stories will be linked to existing sections and activities within
the module.</p>
</li>
<li>
<p>Teachers can select predetermined story themes (e.g. fantastic hero journey,
time-travel quest) or customise their own theme.</p>
</li>
<li>
<p>Teachers can customise the number of game stories and collectibles generated
in Advanced Settings.</p>
</li>
</ol>
</li>
<li>
<p>Learning Assistant (LEA)</p>
<ul data-tight="true" class="tight">
<li>
<p><a href="https://www.learning.moe.edu.sg/teacher-user-guide/collaborate/add-a-discussion/" rel="noopener nofollow" target="_blank">Learning Assistant in Live Chat</a>:</p>
<ol data-tight="true" class="tight">
<li>
<p>Teachers can enable LEA in Live Chat, allowing students to interact with
SALiS during Live Chat discussions.</p>
</li>
</ol>
</li>
<li>
<p><a href="https://www.learning.moe.edu.sg/teacher-user-guide/author/activate-learning-assistant-in-interactive-component/" rel="noopener nofollow" target="_blank">Learning Assistant 1-to-1</a>:</p>
<ol data-tight="true" class="tight">
<li>
<p>Teachers can manually end and close LEA 1-to-1 discussions. The discussion
will be converted into a discussion post and past interactions will be
stored as comments.</p>
</li>
<li>
<p>LEA 1-to-1 can return rich formatting (e.g., bold, lists, tables, line
breaks, special characters).</p>
</li>
</ol>
</li>
</ul>
</li>
<li>
<p>Short Answer Feedback Assistant (ShortAnsFA) Enhancements</p>
<ol data-tight="true" class="tight">
<li>
<p><a href="https://www.learning.moe.edu.sg/teacher-user-guide/assess/add-short-answer-feedback-assistant/" rel="noopener nofollow" target="_blank">Teachers</a> can
provide additional instructions when setting up ShortAnsFA. They can select
predetermined feedback styles (e.g., direct answer, simple feedback, Socratic
feedback) or customise their own instructions.</p>
</li>
<li>
<p>The leniency field will be removed for ShortAnsFA.</p>
</li>
</ol>
</li>
<li>
<p><a href="https://www.learning.moe.edu.sg/teacher-user-guide/organise/create-class-groups-and-communities/" rel="noopener nofollow" target="_blank">Groups</a>
</p>
<ol data-tight="true" class="tight">
<li>
<p>Owners and co-teachers of groups, as well as SLS School Administrators,
can temporarily add a teacher to a group with a set end date. However,
an end date cannot be set for owners so that owners will always retain
access to their groups.</p>
</li>
<li>
<p>When new teachers are added to a group, they will now receive the Observer
role by default, giving them immediate viewing access to assignments.</p>
</li>
<li>
<p>Owners and co-teachers of groups can hide the list of group members from
students for better management of students' privacy.</p>
</li>
</ol>
</li>
<li>
<p>Community Gallery</p>
<ol data-tight="true" class="tight">
<li>
<p>Teachers receive a notification once they submit a module to the Community
Gallery.</p>
</li>
<li>
<p>A consistent URL is now used for Community Gallery modules whether pending
review, featured or returned, so teachers can share links right after submission.</p>
</li>
<li>
<p>Modules pending review or returned now show a placeholder page when accessed
via a shareable URL.</p>
</li>
<li>
<p>All teachers who collaborated on a Community Gallery module will be notified
when their module receives a teacher review.</p>
</li>
</ol>
</li>
<li>
<p>Learning Outcome Display Name</p>
<ol data-tight="true" class="tight">
<li>
<p>Teachers can toggle between viewing displayed learning outcomes or learning
outcomes. Displayed learning outcomes are learning outcomes with more student-friendly
language that are shown to students when added by Content Approvers and
Content Officers (CAs and COs).</p>
</li>
</ol>
</li>
<li>
<p><a href="https://www.learning.moe.edu.sg/teacher-user-guide/organise/create-class-groups-and-communities/" rel="noopener nofollow" target="_blank">QR and Shortened Link Generation</a>
</p>
<ol data-tight="true" class="tight">
<li>
<p>Teachers can generate QR codes with SLS branding for assignments, modules,
groups and community URLs, so students and colleagues can access them quickly.</p>
</li>
<li>
<p>Shortened URLs are now generated automatically and can be copied or used
to create QR codes.</p>
</li>
</ol>
</li>
</ol>
<h2>Features for Students</h2>
<ol data-tight="true" class="tight">
<li>
<p><a href="https://www.learning.moe.edu.sg/my-notebooks/" rel="noopener nofollow" target="_blank">My Notebooks</a>
</p>
<ol data-tight="true" class="tight">
<li>
<p>Students can create notebooks to add and organise notes/annotations from
assignments and resources. These added notes/annotations can be edited
from notebooks directly and edits made will be updated in the original
modules, which can be accessed via links on each note/annotation.</p>
</li>
<li>
<p>Students can also create their own revision notes inside notebooks using
the Rich Text Editor.</p>
</li>
<li>
<p>Students can embed Google Drive files, attach files or add audio recordings
to notes and notebook covers.</p>
</li>
<li>
<p>Students can organise their notes by moving components or pages to specific
positions, creating links to specific notebook pages, or adding coloured
tabs to pages.</p>
</li>
<li>
<p>Students can view and print notebooks in a continuous, print-friendly
format.</p>
</li>
<li>
<p>Students can organise their notebooks by creating categories and moving
notebooks between categories. Students can also toggle between card and
list views of their notebooks.</p>
</li>
</ol>
</li>
<li>
<p>Assignments</p>
<ul data-tight="true" class="tight">
<li>
<p><strong>Assignment Sharing and Study Buddy List</strong>:</p>
<ol data-tight="true" class="tight">
<li>
<p>Students can send and receive study buddy invitations.</p>
</li>
<li>
<p>Students can filter by group or search for peers when adding buddies.</p>
</li>
<li>
<p>Students can view and manage their study buddy list.</p>
</li>
<li>
<p>Students can manage pending approval requests and invite buddies via the
assignment page.</p>
</li>
<li>
<p>Assignments set to shareable by their teachers will be auto-shared when
study buddies are added.</p>
</li>
<li>
<p>Students can @-mention their study buddies in shared assignments and their
buddies will receive notifications.</p>
</li>
</ol>
</li>
<li>
<p><strong>Print-Friendly Worksheet with Responses</strong>:</p>
<ol data-tight="true" class="tight">
<li>
<p>Students can print assignments or modules showing their responses, suggested
answers, teacher feedback, and rubrics if enabled by their teachers.</p>
</li>
</ol>
</li>
</ul>
</li>
<li>
<p>Starred Resources</p>
<ol data-tight="true" class="tight">
<li>
<p>Students can search for starred resources by title on the Starred Resources
page.</p>
</li>
</ol>
</li>
<li>
<p>Adaptive Learning System (ALS) Enhancements</p>
<ol data-tight="true" class="tight">
<li>
<p>Students can select a maximum number of 50 learning outcomes for Test
Myself mode in ALS.</p>
</li>
<li>
<p>Students can receive recommendations drawn from resources (tagged to ALS
content maps) in active groups they belong to when they enter ALS.</p>
</li>
</ol>
</li>
<li>
<p>Learning Assistant in Live Chat</p>
<ol data-tight="true" class="tight">
<li>
<p>Students can use @-mention to call on SALiS during Live Chat discussions
for assistance.</p>
</li>
</ol>
</li>
<li>
<p>Learning Outcome Display Name</p>
<ol data-tight="true" class="tight">
<li>
<p>Students will see displayed learning outcomes in more student-friendly
language if added by Content Approvers and Content Officers (CAs and COs).</p>
</li>
</ol>
</li>
</ol>

</ol>
